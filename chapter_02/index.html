    빛으로 말하기 – RGB LED Stick 프로그래밍과 창작 실습¶

    50분 | RGB 색상 모델 | NeoPixel LED 제어 | 난이도 ★★☆

    학습목표: RGB 색상 모델을 이해하고 (R,G,B) 튜플로 원하는 색을 표현할 수 있다 · NeoPixel 라이브러리로 LED를 개별/전체 제어할 수 있다 · 반복문과 함수를 활용하여 나만의 LED 애니메이션을 창작할 수 있다

    오늘의 질문: "숫자 세 개만으로 어떻게 1600만 가지 색을 만들 수 있을까?"

    🗺️ 전체 흐름에서 오늘의 위치¶

    지난 시간에 MQ2 센서로 세상을 읽는 법(입력)을 배웠습니다. 오늘은 정반대 방향입니다. Pico가 세상에 말하는 법(출력)을 배울 차례입니다!

    flowchart LR A["1차시<br/>🔍 입력<br/>MQ2 센서로<br/>세상 읽기"] --> B["2차시<br/>💡 출력<br/>RGB LED로<br/>빛 표현하기"] B --> C["3차시<br/>🔗 통합<br/>센서+LED<br/>검출 시스템"] style A fill:#e0e0e0,stroke:#999,color:#666 style B fill:#ff9800,stroke:#e65100,color:#fff,stroke-width:3px style C fill:#e0e0e0,stroke:#999,color:#666

    💡 입력(센서)과 출력(LED)은 동전의 양면입니다. 오늘 배운 출력 기술은 3차시에서 센서와 결합되어 완전한 시스템이 됩니다.

    🎨 핵심 개념 1: RGB 색상 모델¶

    빛의 3원색(빨강·초록·파랑)을 섞어 모든 색을 만드는 원리를 이해하고, (R, G, B) 튜플로 색상을 정확히 표현합니다.

    💡 핵심 개념 2: NeoPixel LED 제어¶

    LED 10개를 개별 주소로 제어하는 방법을 익히고, 반복문과 함수를 활용해 다양한 빛 애니메이션을 만듭니다.

    🎭 창작 활동: 나만의 빛 작품¶

    숨쉬기·나이트라이더·경광등 패턴 뼈대를 수정하고 조합하여, 자기만의 LED 조명 작품을 설계합니다.

    1단계: 도입 – 빛의 3원색과 RGB 모델 (5분)¶

    물감 혼합과 빛 혼합의 차이를 비유로 이해하고, (R,G,B) 튜플의 구조를 파악합니다.

    2단계: 실습 1 – LED 기초 제어 (15분)¶

    회로를 연결하고, 전체 LED 색상 변경 → 개별 LED 제어 → 나만의 색 탐색을 단계적으로 실습합니다.

    3단계: 실습 2 – 애니메이션 창작 (20분)¶

    무지개 흐르기 애니메이션을 구현한 뒤, 다양한 패턴 뼈대를 자유롭게 수정하며 나만의 작품을 만듭니다.

    4단계: 정리 – 작품 감상과 성찰 (10분)¶

    교실 조명을 끄고 작품을 발표하며, '같은 하드웨어·다른 소프트웨어'의 의미를 되새깁니다.

    1단계: 빛의 3원색과 RGB 색상 모델 (00~05분)¶

    물감 vs 빛 — 섞으면 정반대!¶

    여러분, 미술 시간에 물감을 섞어본 적 있죠? 빨강 + 파랑 = 보라, 모든 색을 다 섞으면 거의 검정에 가까워집니다. 그런데 빛은 정반대입니다!

    flowchart LR subgraph 물감["🖌️ 물감 혼합 (감산 혼합)"] P1["빨강 물감"] --- PM["섞으면"] P2["파랑 물감"] --- PM P3["초록 물감"] --- PM PM --> PResult["⬛ 검정에 가까움"] end subgraph 빛["💡 빛 혼합 (가산 혼합)"] L1["🔴 빨간 빛"] --- LM["섞으면"] L2["🔵 파란 빛"] --- LM L3["🟢 초록 빛"] --- LM LM --> LResult["⬜ 흰색!"] end style PResult fill:#333,color:#fff style LResult fill:#fff,stroke:#333,color:#333
    비교 항목 물감 (감산 혼합) 빛 (가산 혼합)
    3원색 빨강, 노랑, 파랑 빨강(R), 초록(G), 파랑(B)
    모두 섞으면 검정(어두워짐) 흰색(밝아짐)
    아무것도 없으면 흰색(종이색) 검정(어둠)
    사용처 그림, 인쇄 모니터, LED, 스마트폰

    🔑 핵심 포인트: 컴퓨터 화면, 스마트폰, 그리고 오늘 사용할 RGB LED는 모두 빛의 가산 혼합 방식입니다. 빨강(Red) + 초록(Green) + 파랑(Blue) = 흰색!

    (R, G, B) 튜플이란?¶

    RGB LED의 색상은 숫자 세 개의 묶음으로 표현합니다. 파이썬에서는 이 묶음을 튜플(tuple)이라고 부릅니다.

    (R, G, B) = (빨강 밝기, 초록 밝기, 파랑 밝기)

각 값의 범위: 0(꺼짐) ~ 255(최대 밝기)

    왜 0~255일까? 각 색상 채널이 8비트(2⁸ = 256단계)로 표현되기 때문입니다. 세 채널을 합치면 총 24비트, 즉 256 × 256 × 256 = 16,777,216가지 색을 만들 수 있습니다. 약 1600만 색이죠!

    RGB 값 결과 색상 설명
    (255, 0, 0) 🔴 빨강 R만 최대
    (0, 255, 0) 🟢 초록 G만 최대
    (0, 0, 255) 🔵 파랑 B만 최대
    (255, 255, 0) 🟡 노랑 R + G
    (255, 0, 255) 🟣 보라 R + B
    (0, 255, 255) 🩵 청록 G + B
    (255, 255, 255) ⬜ 흰색 모두 최대
    (0, 0, 0) ⬛ 꺼짐 모두 0
    (255, 165, 0) 🟠 주황 R 최대 + G 중간
    (128, 0, 0) 🟤 어두운 빨강 R 절반

    💡 밝기 조절 팁: 같은 색이라도 숫자를 줄이면 어두워집니다. (255, 0, 0)은 밝은 빨강, (50, 0, 0)은 어두운 빨강. 비율은 유지하면서 전체 값만 줄이세요!

    자기점검 퀴즈

    Q1. 빨간 빛과 초록 빛을 동시에 최대 밝기로 섞으면 무슨 색이 나올까요?

    ① 보라 ② 주황 ③ 노랑 ④ 흰색

    정답 확인

    정답: ③ 노랑

    해설: (255, 255, 0) = 빨강 + 초록 = 노랑입니다. 물감에서는 빨강+초록=갈색이지만, 빛에서는 노랑이 됩니다! 보라는 R+B, 흰색은 R+G+B 모두 섞어야 합니다. 잘 맞혔다면 가산 혼합 개념을 잘 이해한 거예요! 👏

    Q2. (100, 0, 50)은 어떤 색에 가까울까요?

    ① 밝은 하늘색 ② 어두운 분홍/자주 ③ 연두색 ④ 밝은 노랑

    정답 확인

    정답: ② 어두운 분홍/자주

    해설: R=100(중간), G=0(없음), B=50(약간)이므로 빨강 계열에 파랑이 살짝 섞인 어두운 분홍/자주색입니다. G가 0이므로 초록 계열 색은 나올 수 없습니다. 틀렸다면 위의 RGB 색상 표를 다시 살펴보세요!

    2단계: 실습 1 – LED 기초 제어 (05~20분)¶

    NeoPixel이란?¶

    RGB LED Stick에는 WS2813 칩이 내장된 LED 10개가 일렬로 배치되어 있습니다. 이 LED들은 NeoPixel 프로토콜이라는 방식으로 통신합니다.

    sequenceDiagram participant Pico as Pico (GP16) participant LED0 as LED #0 participant LED1 as LED #1 participant LED9 as LED #9 Note over Pico: np.write() 호출! Pico->>LED0: LED#0 색상 데이터<br/>(R, G, B) LED0->>LED1: LED#1 색상 데이터<br/>(R, G, B) LED1-->>LED9: ... 순서대로 전달 ... Note over LED0,LED9: 모든 LED가<br/>동시에 점등!
    NeoPixel 특징 설명
    데이터 선 1개 GP16 핀 하나로 10개 LED를 모두 제어
    개별 주소 지정 np[0], np[1], ..., np[9]로 각각 다른 색 가능
    직렬 전송 데이터가 LED0 → LED1 → ... → LED9 순서로 전달됨
    write() 필수 색상을 넣은 후 np.write()를 호출해야 실제 LED에 반영

    회로 연결¶

    Grove LED Stick을 Shield의 D16 포트에 꽂으세요. 연결은 이게 전부입니다!

    Grove Shield 포트 Pico 핀 연결 대상 역할
    D16 GP16 RGB LED Stick (IN 쪽) NeoPixel 데이터 전송
    VCC (3.3V) - LED 전원 자동 연결
    GND - LED 접지 자동 연결
    flowchart LR subgraph Shield["Grove Shield (Pico 장착)"] D16["D16 포트"] end subgraph LED["RGB LED Stick"] IN["IN 쪽<br/>⚠️ 이쪽을 연결!"] L0["LED 0"] L1["LED 1"] L9["LED 9"] OUT["OUT 쪽"] IN --> L0 --> L1 --> L9 --> OUT end D16 -->|"Grove 케이블"| IN style IN fill:#ff5722,color:#fff,stroke-width:2px style OUT fill:#9e9e9e,color:#fff

    ⚠️ 주의! LED Stick에는 IN(입력)과 OUT(출력) 방향이 있습니다. IN 쪽이 Grove Shield에 연결되어야 합니다. 거꾸로 꽂으면 LED가 동작하지 않습니다. 표시가 작으니 잘 확인하세요!

    Step 1: 전체 LED 한 색으로 켜기 (v1 — 가장 기본)¶

    먼저 가장 간단한 코드부터 시작합니다. LED 10개를 한 가지 색으로 켜 봅시다.

    # === step1_v1: 가장 기본 - LED 전체 빨간색 켜기 ===
from machine import Pin
from neopixel import NeoPixel

# LED 설정: GP16 핀, LED 10개
np = NeoPixel(Pin(16), 10)

# 10개 LED를 모두 빨간색으로 설정
for i in range(10):
    np[i] = (255, 0, 0)  # <- (R, G, B) 튜플로 색 지정

# 반드시 write()를 호출해야 LED에 반영!
np.write()  # <- 이걸 빼먹으면 아무 일도 안 일어남

    예상 결과: LED 10개가 모두 밝은 빨간색으로 켜집니다.

    위 코드의 핵심은 두 줄입니다: - 3번째 줄 np = NeoPixel(Pin(16), 10) — GP16 핀에 연결된 LED 10개를 제어할 객체 생성 - 7번째 줄 np[i] = (255, 0, 0) — i번째 LED에 RGB 색상을 지정 (리스트 인덱싱과 같은 문법!) - 10번째 줄 np.write() — 설정한 색상을 실제 LED에 전송 (이게 바로 NeoPixel 데이터 전송)

    🐛 에러 경험: write()를 빼먹으면?¶

    아래 코드를 실행하면 어떻게 될까요?

    # ❌ 의도적으로 잘못된 코드 — 실행해보세요!
from machine import Pin
from neopixel import NeoPixel

np = NeoPixel(Pin(16), 10)

for i in range(10):
    np[i] = (0, 255, 0)  # 초록색 설정

# np.write()  ← 이 줄이 빠져있음!
print("코드 실행 완료!")
    결과 확인 **결과**: 셸에 "코드 실행 완료!"는 출력되지만, **LED는 아무 변화 없음**! **원인**: `np[i] = (0, 255, 0)`은 Pico 내부의 메모리에 색상 정보를 저장할 뿐, 실제 LED로 데이터를 보내지 않습니다. `np.write()`가 호출되어야 비로소 저장된 데이터가 LED Stick으로 전송됩니다. **기억하세요**: 색 넣기 → **반드시 `np.write()`** → LED 반영!

    Step 1: 색상 퍼레이드 (v2 — 기능 추가)¶

    이전 코드에서 fill 함수와 색상 사전을 추가하여, 여러 색이 순서대로 바뀌는 코드로 업그레이드합니다.

    # === step1_v2: 색상 퍼레이드 ===
# 이전 코드에서 달라진 점: fill() 함수 + 여러 색 순환 + time 모듈 추가
from machine import Pin
from neopixel import NeoPixel
import time  # <- 추가! 시간 지연용

NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS)

# --- 색상 사전: 자주 쓰는 색을 이름으로 저장 ---
RED    = (255, 0, 0)
GREEN  = (0, 255, 0)
BLUE   = (0, 0, 255)
YELLOW = (255, 255, 0)
PURPLE = (128, 0, 128)
CYAN   = (0, 255, 255)
WHITE  = (255, 255, 255)
OFF    = (0, 0, 0)

# --- 전체 LED를 한 색으로 채우는 함수 ---
def fill(color):
    """LED 전체를 같은 색으로 설정하고 적용"""
    for i in range(NUM_LEDS):
        np[i] = color
    np.write()  # <- fill 안에 write()를 포함시켜서 실수 방지!

# --- 색상 퍼레이드 실행 ---
colors = [RED, GREEN, BLUE, YELLOW, PURPLE, CYAN, WHITE]
names  = ["빨강", "초록", "파랑", "노랑", "보라", "청록", "흰색"]

for color, name in zip(colors, names):
    fill(color)
    print(f"{name} 점등!")
    time.sleep(1)  # <- 1초 대기 후 다음 색으로

fill(OFF)  # 모든 LED 끄기
print("끝!")

    # === step2: 개별 LED 제어 + 색상 실험 ===

for i in range(NUM_LEDS):
    # 먼저 전체 끄기
    for j in range(NUM_LEDS):
        np[j] = (0, 0, 0)
    # i번째 LED만 파란색으로 켜기

np.write()
time.sleep(3)

    Q1. 다음 코드를 실행하면 LED Stick은 어떻게 보일까요? 

    for i in range(10):
    np[i] = (0, 0, 0)
np[5] = (255, 255, 0)
np.write()

    ① 10개 모두 노란색 ② 6번째 LED만 노란색, 나머지 꺼짐 ③ 5개는 노란색, 5개는 꺼짐 ④ 아무 일도 안 일어남

    정답 확인

    정답: ② 6번째 LED만 노란색, 나머지 꺼짐

    해설: np[5]는 인덱스 5, 즉 6번째 LED입니다 (0부터 세니까요!). 나머지는 (0,0,0)으로 꺼져 있고, (255,255,0)은 R+G = 노란색입니다. np.write()가 있으므로 정상 반영됩니다. 인덱스가 0부터 시작한다는 점을 잊지 마세요!

    Q2. (255, 0, 0)의 절반 밝기 빨강을 만들려면 어떻게 해야 할까요?

    ① (128, 0, 0) ② (255, 128, 128) ③ (128, 128, 128) ④ (255, 0, 0, 128)

    정답 확인

    정답: ① (128, 0, 0)

    해설: 밝기를 줄이려면 비율은 유지하면서 모든 값을 같은 비율로 줄입니다. R만 있는 빨강이므로 R만 반으로 줄이면 됩니다. ②는 분홍색, ③은 회색, ④는 튜플 원소가 4개라 오류가 납니다.

    3단계: 실습 2 – 애니메이션 창작 (20~40분)¶

    wheel 함수 — 숫자 하나로 무지개 전체를!¶

    무지개를 만들려면 RGB 값을 부드럽게 변환해야 합니다. 이때 사용하는 것이 wheel 함수입니다. 0~255 사이의 숫자 하나를 넣으면 무지개 위의 해당 위치 색상이 (R,G,B)로 나옵니다.

    Step 3: 무지개 흐르기 (v1 → v2)¶

    v1: wheel 함수만 먼저 이해하기

    # === step3_v1: wheel 함수 테스트 ===
# 10개 LED에 무지개를 한 번 표시하는 코드
from machine import Pin
from neopixel import NeoPixel

NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS)

def wheel(pos):
    """0~255 → 무지개 색상 (R,G,B) 변환"""
    if pos < 85:
        return (255 - pos * 3, pos * 3, 0)
    elif pos < 170:
        pos -= 85
        return (0, 255 - pos * 3, pos * 3)
    else:
        pos -= 170
        return (pos * 3, 0, 255 - pos * 3)

# LED 10개에 무지개 배치 (정지 상태)
for i in range(NUM_LEDS):
    color_pos = i * 256 // NUM_LEDS  # 0, 25, 51, 76, ...
    np[i] = wheel(color_pos)
np.write()
print("무지개 정지 상태!")

    예상 결과: LED 10개에 빨강→주황→노랑→초록→파랑→보라 순으로 무지개가 표시됩니다 (움직이지 않음).

    v2: 애니메이션 추가 — 무지개가 흐른다!

    v1에서 달라진 점: offset 변수를 증가시키면서 무지개가 이동하는 효과를 만듭니다.

    # === step3_v2: 무지개 흐르기 애니메이션 ===
# v1에서 달라진 점: offset으로 무지개가 흘러가는 효과 추가
from machine import Pin
from neopixel import NeoPixel
import time  # <- 추가! 애니메이션 속도 조절

NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS)

def wheel(pos):
    """0~255 → 무지개 색상 (R,G,B) 변환"""
    if pos < 85:
        return (255 - pos * 3, pos * 3, 0)
    elif pos < 170:
        pos -= 85
        return (0, 255 - pos * 3, pos * 3)
    else:
        pos -= 170
        return (pos * 3, 0, 255 - pos * 3)

def rainbow_flow(wait=0.05):
    """무지개가 LED 위를 흘러가는 애니메이션"""
    for offset in range(256):       # offset: 0→255로 증가
        for i in range(NUM_LEDS):
            # 각 LED마다 위치를 다르게 계산 → 무지개 배치
            color_pos = (i * 256 // NUM_LEDS + offset) % 256
            #                                    ↑ % 256: 255 넘으면 다시 0으로!
            np[i] = wheel(color_pos)
        np.write()
        time.sleep(wait)  # <- wait 값으로 속도 조절

# --- 무지개 3바퀴! ---
print("무지개 시작!")
for _ in range(3):
    rainbow_flow(0.03)  # 0.03초 간격 (빠르게!)

# 끄기
for i in range(NUM_LEDS):
    np[i] = (0, 0, 0)
np.write()
print("끝!")

    예상 출력: 

    무지개 시작!
끝!

    (LED에서는 무지개가 3바퀴 흘러간 뒤 꺼집니다)

    핵심 원리 분해:

    코드 역할
    offset 0→255로 증가하면서 무지개 전체를 한 칸씩 밀어냄
    i * 256 // NUM_LEDS LED마다 무지개 위의 다른 위치에 배치
    + offset 프레임마다 위치를 이동시켜 "흐르는" 효과
    % 256 나머지 연산 — 255를 넘으면 다시 0으로 돌아가 무한 순환
    time.sleep(wait) 프레임 간격 — 작을수록 빠르게, 클수록 느리게

    🎮 속도를 바꿔보세요! rainbow_flow(0.01) → 매우 빠름, rainbow_flow(0.1) → 느긋하게

    아래 세 가지 패턴 뼈대를 제공합니다. 색상, 속도, 반복 횟수를 자유롭게 수정하여 나만의 작품을 만들어보세요!

    # === step4: 나만의 LED 패턴 만들기 ===
# 패턴 A/B/C 중 원하는 것을 골라 수정하세요!
from machine import Pin
from neopixel import NeoPixel
import time

NUM_LEDS = 10
np = NeoPixel(Pin(16), NUM_LEDS)

# ── 패턴 A: 숨쉬기 (한 색이 천천히 밝아졌다 어두워짐) ──
def breathe(r, g, b, speed=0.03):
    """숨쉬기 효과 — r,g,b에 원하는 색 비율 입력"""
    # 밝아지기: 0% → 100%
    for brightness in range(0, 100, 2):
        scale = brightness / 100  # 0.0 → 0.98
        for i in range(NUM_LEDS):
            np[i] = (int(r*scale), int(g*scale), int(b*scale))
            #        ↑ int(): 소수점 버림 (RGB는 정수만 가능!)
        np.write()
        time.sleep(speed)
    # 어두워지기: 100% → 0%
    for brightness in range(100, 0, -2):
        scale = brightness / 100
        for i in range(NUM_LEDS):
            np[i] = (int(r*scale), int(g*scale), int(b*scale))
        np.write()
        time.sleep(speed)

# ── 패턴 B: 나이트라이더 (불빛이 좌우로 왔다갔다) ──
def knight_rider(r, g, b, speed=0.08):
    """좌우 왕복 효과 — 잔상 포함"""
    # 오른쪽으로 이동
    for i in range(NUM_LEDS):
        for j in range(NUM_LEDS):
            np[j] = (0, 0, 0)         # 전체 끄기
        np[i] = (r, g, b)              # 메인 불빛
        if i > 0:
            np[i-1] = (r//4, g//4, b//4)   # 왼쪽 잔상
            #          ↑ r//4: 정수 나눗셈으로 밝기 1/4
        if i < NUM_LEDS - 1:
            np[i+1] = (r//4, g//4, b//4)   # 오른쪽 잔상
        np.write()
        time.sleep(speed)
    # 왼쪽으로 이동 (되돌아옴)
    for i in range(NUM_LEDS - 2, 0, -1):
        for j in range(NUM_LEDS):
            np[j] = (0, 0, 0)
        np[i] = (r, g, b)
        if i > 0:
            np[i-1] = (r//4, g//4, b//4)
        if i < NUM_LEDS - 1:
            np[i+1] = (r//4, g//4, b//4)
        np.write()
        time.sleep(speed)

# ── 패턴 C: 경광등 (빨강/파랑 교대) ──
def police_light(speed=0.15):
    """경광등 효과 — 좌우 반반 깜빡임"""
    for _ in range(5):
        # 왼쪽 반 빨강, 오른쪽 반 파랑
        for i in range(5):
            np[i] = (255, 0, 0)
        for i in range(5, 10):
            np[i] = (0, 0, 255)
        np.write()
        time.sleep(speed)
        # 색상 반전!
        for i in range(5):
            np[i] = (0, 0, 255)
        for i in range(5, 10):
            np[i] = (255, 0, 0)
        np.write()
        time.sleep(speed)

# =============================================
#  🎨 아래에서 원하는 패턴을 실행하세요!
#     색상과 속도를 자유롭게 바꿔보세요.
# =============================================

# 패턴 A 실행: 하늘색 숨쉬기 (색상 바꿔보세요!)
for _ in range(3):
    breathe(0, 100, 255)

# 패턴 B 실행: 빨간 나이트라이더 (색상 바꿔보세요!)
for _ in range(3):
    knight_rider(255, 0, 0)

# 패턴 C 실행: 경광등 (속도 바꿔보세요!)
police_light(0.1)

# 전체 끄기
for i in range(NUM_LEDS):
    np[i] = (0, 0, 0)
np.write()
print("나의 조명 쇼 끝!")

    각 패턴의 핵심 기법:

    패턴 핵심 기법 커스터마이징 포인트
    A: 숨쉬기 scale로 밝기 비율 조절, int()로 정수 변환 색상 (r,g,b), 속도 speed
    B: 나이트라이더 개별 LED 인덱싱, //4로 잔상 효과 색상, 속도, 잔상 비율(//4 → //2로 바꾸면?)
    C: 경광등 범위 분할(range(5), range(5,10)), 색상 반전 속도, 분할 위치, 색상 조합

    힌트 `police_light` 함수 안에서 `(255, 0, 0)`을 초록 `(0, 255, 0)`으로, `(0, 0, 255)`를 노랑 `(255, 255, 0)`으로 바꾸면 됩니다.
    정답 코드 
    def traffic_light(speed=0.15):
    """초록/노랑 교대 깜빡임"""
    for _ in range(5):
        for i in range(5):
            np[i] = (0, 255, 0)      # 왼쪽: 초록
        for i in range(5, 10):
            np[i] = (255, 255, 0)    # 오른쪽: 노랑
        np.write()
        time.sleep(speed)
        for i in range(5):
            np[i] = (255, 255, 0)    # 반전
        for i in range(5, 10):
            np[i] = (0, 255, 0)
        np.write()
        time.sleep(speed)

    응용: 새 상황

    숨쉬기 패턴에서 LED마다 시간차를 주어 '파도' 효과를 만들어보세요. (힌트: LED 번호 i에 따라 brightness 시작점을 다르게!)

    힌트 각 LED의 밝기를 계산할 때 `(brightness + i * 10) % 100`처럼 LED 번호에 따른 오프셋을 주면 파도처럼 보입니다.
    정답 코드 
    def wave(r, g, b, speed=0.05):
    """파도 효과 — LED마다 시간차가 있는 숨쉬기"""
    for step in range(100):
        for i in range(NUM_LEDS):
            # 각 LED에 다른 오프셋 적용
            brightness = (step + i * 10) % 100
            # 삼각파: 0→50→0으로 밝기 변화
            if brightness > 50:
                brightness = 100 - brightness
            scale = brightness / 50
            np[i] = (int(r*scale), int(g*scale), int(b*scale))
        np.write()
        time.sleep(speed)

# 실행
for _ in range(3):
    wave(0, 150, 255)  # 하늘색 파도

    도전: 결합/설계

    LED가 양쪽 끝에서 가운데로 모이다가, 가운데에서 다시 양쪽으로 퍼지는 패턴 함수 converge(r, g, b)를 직접 설계해보세요.

    힌트 왼쪽 인덱스 `left`와 오른쪽 인덱스 `right`를 동시에 움직이세요. `left`는 0→4, `right`는 9→5로 이동하면 가운데서 만납니다.
    정답 코드 
    def converge(r, g, b, speed=0.12):
    """양쪽에서 가운데로 모였다 퍼지는 효과"""
    # 모이기: 양쪽 끝 → 가운데
    for step in range(5):
        for j in range(NUM_LEDS):
            np[j] = (0, 0, 0)
        np[step] = (r, g, b)              # 왼쪽에서
        np[NUM_LEDS - 1 - step] = (r, g, b)  # 오른쪽에서
        np.write()
        time.sleep(speed)
    # 퍼지기: 가운데 → 양쪽 끝
    for step in range(4, -1, -1):
        for j in range(NUM_LEDS):
            np[j] = (0, 0, 0)
        np[step] = (r, g, b)
        np[NUM_LEDS - 1 - step] = (r, g, b)
        np.write()
        time.sleep(speed)

# 실행
for _ in range(3):
    converge(255, 0, 255)  # 보라색 수렴

    형성 평가¶

    [개념 확인] Q1. RGB 색상 모델에서 (0, 255, 255)는 무슨 색인가요?

    ① 보라 ② 노랑 ③ 청록(시안) ④ 주황

    정답 확인 **정답: ③ 청록(시안)** **해설**: R=0(빨강 없음), G=255(초록 최대), B=255(파랑 최대)이므로 초록+파랑=청록(Cyan)입니다. 노랑은 (255,255,0), 보라는 (128,0,128) 또는 (255,0,255), 주황은 (255,165,0)입니다. 잘 맞혔다면 RGB 혼합을 완벽히 이해한 거예요! 👏

    [개념 확인] Q2. NeoPixel LED 제어에서 반드시 호출해야 하는 함수는?

    ① np.show() ② np.update() ③ np.write() ④ np.send()

    정답 확인 **정답: ③ np.write()** **해설**: MicroPython의 NeoPixel 라이브러리에서 `np.write()`를 호출해야 설정한 색상 데이터가 실제 LED로 전송됩니다. 이걸 빼먹으면 Pico 메모리에만 저장되고 LED는 변화 없습니다. Arduino 라이브러리에서는 `show()`를 쓰지만, MicroPython에서는 `write()`입니다.

    [응용] Q3. 다음 코드의 빈칸을 채워 LED 0~4번을 빨강, 5~9번을 파랑으로 만드세요.

    for i in range(10):
    if i < 5:
        np[i] = (_____, _____, _____)
    else:
        np[i] = (_____, _____, _____)
np.write()
    정답 확인 
    for i in range(10):
    if i < 5:
        np[i] = (255, 0, 0)   # 빨강
    else:
        np[i] = (0, 0, 255)   # 파랑
np.write()
    **해설**: LED 인덱스 0~4는 `if i < 5` 조건에 해당하여 빨강 `(255, 0, 0)`, 5~9는 `else`에 해당하여 파랑 `(0, 0, 255)`이 됩니다. 이 구조가 바로 경광등 패턴의 기본 원리입니다!

    [응용] Q4. wheel(0), wheel(85), wheel(170)은 각각 어떤 색을 반환하나요? 순서대로 짝을 지으세요.

    함수 호출 보기
    A. wheel(0) ㄱ. 초록
    B. wheel(85) ㄴ. 파랑
    C. wheel(170) ㄷ. 빨강
    정답 확인 **A-ㄷ, B-ㄱ, C-ㄴ** - `wheel(0)` → (255, 0, 0) = 빨강 - `wheel(85)` → (0, 255, 0) = 초록 - `wheel(170)` → (0, 0, 255) = 파랑 **해설**: wheel 함수는 0~84 구간에서 빨강→초록, 85~169 구간에서 초록→파랑, 170~255 구간에서 파랑→빨강으로 변환합니다. 각 구간의 시작점이 바로 빨강, 초록, 파랑입니다.

    [심화/서술] Q5. 학교 복도에 설치할 비상구 안내 LED 시스템을 설계한다고 가정합니다. 평상시, 대피 시작, 긴급 상황 세 가지 상태를 색상·밝기·속도 조합으로 어떻게 표현하겠습니까? 오늘 배운 패턴을 활용하여 설명하세요.

    예시 답안 **평상시**: 🟢 초록색 고정, 낮은 밝기 `(0, 30, 0)` — 눈에 거슬리지 않게 은은하게 비상구 위치만 표시 **대피 시작**: 🟢 초록색 숨쉬기(breathe) 패턴, 중간 밝기, 느린 속도(speed=0.05) — "이쪽으로 이동하세요"라는 메시지를 부드럽게 전달 **긴급 상황**: 🔴 빨강 빠른 깜빡임(blink), 최대 밝기 `(255, 0, 0)`, 매우 빠른 속도(speed=0.05) — 즉시 주의를 끌어 긴급 대피를 알림 **설계 원리**: 안정 → 주의 유도 → 긴급 순으로 색상(초록→빨강), 밝기(낮음→높음), 속도(고정→느림→빠름)를 점진적으로 강화하여 사용자가 상황의 심각성을 직관적으로 파악할 수 있도록 합니다.
