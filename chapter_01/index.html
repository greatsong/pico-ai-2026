    학습목표: 1. 아날로그 신호와 디지털 신호의 차이를 일상 비유로 설명할 수 있다 2. ADC를 사용하여 MQ2 가스 센서 값을 읽는 MicroPython 코드를 작성할 수 있다 3. Thonny 플로터와 이동 평균 필터를 활용하여 센서 데이터를 실시간 시각화할 수 있다

    📚 핵심 개념 카드¶

    🔑 핵심 개념 1: 아날로그 vs 디지털 신호¶

    세상의 대부분은 아날로그(연속적으로 변하는 값)입니다. 온도, 소리, 가스 농도… 모두 단계 없이 부드럽게 변하죠. 반면 컴퓨터는 디지털(0과 1, 끊어진 숫자)만 이해합니다. 이 두 세계를 이어주는 다리가 바로 ADC입니다.

    🔑 핵심 개념 2: ADC (Analog-to-Digital Converter)¶

    아날로그 값을 디지털 숫자로 바꿔주는 변환기입니다. Pico 내부에 내장되어 있으며, 16비트 해상도로 아날로그 값을 0~65,535 사이의 정수로 변환합니다. 2^16 = 65,536개의 단계로 세상을 읽는 거예요!

    🛠️ 오늘의 도구: MQ2 가스 센서¶

    내부에 히터와 금속산화물 감지층이 있어서, 공기 중의 가연성 가스(LPG, 알코올, 연기 등)가 감지층에 닿으면 전기 저항이 변합니다. 이 저항 변화를 전압으로 변환해 Pico의 ADC가 읽습니다.

    💬 바이브 코딩 철학¶

    이 수업에서 여러분은 코드를 외우는 사람이 아니라, 문제를 설계하는 사람입니다. 코드는 도구일 뿐이에요. "어떤 문제를 풀고 싶은가?", "센서가 알려주는 숫자를 어떻게 해석할 것인가?"가 더 중요합니다. AI 시대에 필요한 건 코드 암기가 아니라 문제 정의 능력입니다.

    🎬 동기유발 — "이 교실의 공기는 안전할까?"¶

    여러분, 지금 이 교실의 공기는 깨끗할까요? 눈에 보이지 않으니 알 수 없죠. 그런데 만약 가스레인지에서 가스가 새고 있다면? 옆집에서 연기가 올라온다면? 눈에 안 보이는 위험을 어떻게 알아챌 수 있을까요?

    바로 센서입니다. 오늘 우리가 사용할 MQ2 가스 센서는, 공기 중에 떠다니는 가연성 가스 분자를 감지해서 전기 신호로 바꿔줍니다. 이 신호를 Pico가 읽으면, "지금 이 공간의 가스 농도는 얼마"라고 숫자로 알려주는 것이죠.

    💡 생각해보기: 가스 탐지기, 미세먼지 측정기, 체온계… 이 모든 것의 공통점은? 센서가 보이지 않는 물리량을 숫자로 바꿔준다는 것입니다!

    아날로그 신호 vs 디지털 신호¶

    두 종류의 온도계를 떠올려봅시다.

    구분 수은 온도계 (아날로그) 디지털 온도계 (디지털)
    표시 방식 수은이 연속적으로 올라감 숫자가 뚝뚝 끊어져 표시
    36.5℃와 36.6℃ 사이 무한히 많은 중간 값이 존재 36.5 아니면 36.6
    비유 미끄럼틀 (매끄럽게 연결) 계단 (한 칸씩 끊김)
    flowchart LR subgraph 아날로그["🌊 아날로그 신호"] A1["연속적으로 변함<br/>예: 가스 농도, 온도, 소리"] end subgraph 디지털["📊 디지털 신호"] D1["끊어진 값 (0 또는 1)<br/>예: 버튼 ON/OFF, 스위치"] end subgraph ADC변환["⚡ ADC 변환기"] C1["아날로그 → 디지털<br/>연속값을 0~65535로<br/>16비트 해상도"] end 아날로그 -->|"Pico 내장 ADC"| ADC변환 ADC변환 --> 디지털

    핵심 포인트:

    • 버튼을 누르면? → 0 아니면 1. 중간이 없습니다. 이것이 디지털.
    • 가스 농도는? → 조금 있을 수도, 많이 있을 수도, 아주 많이 있을 수도 있죠. 값이 부드럽게 이어집니다. 이것이 아날로그.
    • 문제: 컴퓨터(Pico)는 디지털(숫자)만 이해합니다!
    • 해결: Pico 안에 내장된 ADC가 아날로그 값을 숫자로 변환해줍니다.

    ADC — 아날로그를 디지털로 바꾸는 변환기¶

    ADC는 Analog-to-Digital Converter의 줄임말입니다.

    Pico 2 WH의 ADC는 16비트 해상도를 가집니다. 이것이 의미하는 것은:

    항목 값
    비트 수 16비트
    가능한 단계 수 2^16 = 65,536단계
    출력 범위 0 ~ 65,535
    0일 때 전압 0V (가스 없음에 가까움)
    65,535일 때 전압 3.3V (최대 감지)

    🎓 왜 65,535일까? 2의 16승은 65,536입니다. 0부터 세니까 0, 1, 2, … 65,535. 총 65,536개의 단계로 아날로그 세계를 잘게 쪼개는 거예요. 단계가 많을수록 더 정밀하게 측정할 수 있습니다.

    MQ2 가스 센서의 작동 원리¶

    MQ2 센서 내부에는 두 가지 핵심 부품이 있습니다:

    flowchart TD subgraph MQ2["MQ2 센서 내부"] H["🔥 히터<br/>센서를 200~300℃로 가열"] --> S["🧪 금속산화물 감지층<br/>(SnO₂)"] end G["💨 가스 분자<br/>(LPG, 알코올, 연기)"] --> S S -->|"가스 농도에 따라<br/>저항이 변함"| V["⚡ 전압 변화"] V -->|"ADC가 읽음"| N["📊 0~65535 숫자"]
    1. 히터: 센서 내부를 200~300℃로 가열합니다. 이래야 금속산화물이 가스에 반응할 수 있어요.
    2. 금속산화물 감지층(SnO₂): 가연성 가스 분자가 이 층에 닿으면 전기 저항이 변합니다.
    3. 저항이 변하면 → 전압이 변하고 → ADC가 그 전압을 0~65,535 사이 숫자로 변환합니다.

    ⚠️ 중요! 히터 때문에 MQ2는 전원을 켜고 1~2분 예열이 필요합니다. 처음에 값이 이상하게 나와도 정상이에요! 예열이 끝나면 안정됩니다.

    MQ2가 잘 감지하는 것 MQ2가 잘 못 감지하는 것
    LPG (가정용 가스) 이산화탄소(CO₂)
    알코올 (에탄올) 산소(O₂)
    연기, 수소 질소(N₂)
    프로판, 메탄 수증기(약간 반응)

    자기점검 퀴즈 — 개념 확인

    Q1. 가스 센서가 감지한 농도 값은 연속적으로 변합니다. 이런 신호를 무엇이라 하나요?

    ① 디지털 신호 ② 아날로그 신호 ③ 바이너리 신호 ④ 펄스 신호

    정답 확인

    정답: ②

    해설: 연속적으로 변하는 신호는 아날로그 신호입니다. 디지털 신호는 0과 1처럼 끊어진 값만 가지죠. 가스 농도, 온도, 소리 크기 등 자연계의 대부분이 아날로그입니다. 잘 맞혔다면 개념을 정확히 이해한 거예요! 👏

    Q2. Pico의 ADC는 16비트 해상도를 가집니다. read_u16()으로 읽을 수 있는 최대값은?

    ① 1023 ② 4095 ③ 65535 ④ 16777215

    정답 확인

    정답: ③

    해설: 16비트이므로 2^16 = 65,536. 0부터 시작하니까 최대값은 65,535입니다. ①은 10비트(2^10 - 1), ②는 12비트(2^12 - 1), ④는 24비트(2^24 - 1)입니다. 비트 수가 클수록 더 정밀하게 측정할 수 있어요!

    자기점검 퀴즈 — 코드 이해

    Q3. 아래 코드에서 read_u16()의 "u16"이 의미하는 것은?

    ① USB 16번 포트 ② unsigned 16-bit (부호 없는 16비트) ③ update 16회 ④ unit 16

    정답 확인

    정답: ②

    해설: u는 unsigned(부호 없음), 16은 16비트를 의미합니다. 부호가 없으니 음수는 없고, 0부터 2^16 - 1 = 65,535까지의 양수만 반환합니다. ①③④는 관련 없는 용어예요.

    v3 — 이동 평균 필터 적용 (step2_mq2_plotter.py)¶

    이전 v2 코드에서 이동 평균 함수를 추가합니다:

    # === MQ2 가스 센서 + 이동 평균 필터 + 플로터 시각화 ===
# 10번 읽어서 평균 → 노이즈를 줄여 부드러운 데이터 출력

from machine import ADC, Pin
import time

# --- 센서 설정 ---
gas_sensor = ADC(Pin(26))

# --- [NEW!] 이동 평균 필터 함수 ---
# 왜 필요한가? 센서 노이즈를 줄여서 진짜 변화만 보려고!
def read_average(sensor, count=10):
    """count번 읽어서 평균을 구하는 함수"""
    total = 0                         # <- 합계 변수
    for i in range(count):            # <- count번 반복
        total += sensor.read_u16()    # <- 값을 누적
        time.sleep_ms(10)             # <- 10ms 간격으로 읽기
    return total // count             # <- 정수 나눗셈으로 평균

# --- 메인 루프 ---
while True:
    # [CHANGED!] 1번 읽기 → 10번 읽어서 평균
    avg_value = read_average(gas_sensor, 10)

    # Thonny 플로터용 출력 (숫자만 print하면 자동 그래프!)
    print(avg_value)

    time.sleep(0.5)  # [CHANGED!] 1초 → 0.5초 (더 자주 측정)

    항목 v2 (step1) v3 (step2)
    읽기 방식 1번 읽기 10번 읽어서 평균
    핵심 추가 없음 read_average() 함수
    측정 간격 1초 0.5초 (더 빠른 반응)
    그래프 모양 들쭉날쭉 부드러움

    코드 핵심 해부:

    • 4번째 줄의 def read_average(sensor, count=10):이 바로 함수 정의입니다. count=10은 기본값으로, 호출 시 생략하면 10번 읽습니다.
    • total // count에서 //는 정수 나눗셈입니다. 12,345 ÷ 10 = 1234.5인데, //를 쓰면 1234가 됩니다. 센서 값은 정수여야 하니까요.
    • time.sleep_ms(10)은 밀리초(ms) 단위 대기입니다. time.sleep(0.01)과 거의 같지만, 더 정밀하게 지정할 수 있어요.

    연습 문제¶

    🟢 기초: read_average의 count를 5로 바꿔서 실행해보세요. 10일 때와 비교하면? **힌트**: `read_average(gas_sensor, 5)` 로 바꾸면 됩니다. **정답 코드:** 
    avg_value = read_average(gas_sensor, 5)  # 10 → 5로 변경
    **관찰 결과**: count=5일 때 그래프가 count=10보다 약간 더 들쭉날쭉합니다. 하지만 반응 속도는 더 빠릅니다. 평균 횟수가 적으면 노이즈가 덜 줄어들지만, 변화에 더 빨리 반응하는 **속도-정확도 트레이드오프**가 있습니다!
    🟡 응용: 센서값과 함께 "높음/보통/낮음" 텍스트를 같이 출력하려면? **힌트**: `if-elif-else` 조건문을 사용하세요. **정답 코드:** 
    while True:
    avg_value = read_average(gas_sensor, 10)

    # 조건문으로 상태 판단
    if avg_value > 30000:
        level = "높음"
    elif avg_value > 15000:
        level = "보통"
    else:
        level = "낮음"

    print(avg_value)  # 플로터용 (숫자만!)
    # print(f"값: {avg_value} | 상태: {level}")  # 텍스트 확인용
    time.sleep(0.5)
    **주의**: Thonny 플로터는 숫자만 인식합니다. 텍스트가 섞이면 그래프가 안 그려져요! 텍스트 출력은 확인할 때만 주석을 풀어서 사용하세요.
    🔴 도전: 측정 간격(sleep)을 0.1초로 바꾸고, count도 20으로 올리면 어떻게 될까? **힌트**: 10번 읽는 데 약 100ms, 20번이면 약 200ms가 걸립니다. **정답 코드:** 
    while True:
    avg_value = read_average(gas_sensor, 20)  # 20번 평균
    print(avg_value)
    time.sleep(0.1)  # 0.1초 간격
    **분석**: count=20이면 더 부드러워지지만, 한 번 평균 내는 데 약 200ms가 걸립니다. `sleep(0.1)` = 100ms를 더하면 총 약 300ms마다 한 번 출력됩니다. 그래프는 매우 부드럽지만, 급격한 변화(입김 불기 등)에 대한 **반응이 느려집니다**. 실시간성과 정확성 사이의 균형을 찾는 것이 엔지니어의 중요한 역할이에요!

    탐구 미션에서 가장 인상적이었던 발견을 짧게 공유해봅시다.

    여러분이 오늘 한 것을 정리하면:

    1. 눈에 보이지 않는 가스를 MQ2 센서로 감지했습니다
    2. ADC가 아날로그 값을 0~65,535 숫자로 바꿔주었습니다
    3. 그 숫자를 Thonny 플로터로 그래프로 시각화했습니다
    4. 이동 평균 필터로 노이즈를 줄여 더 깨끗한 데이터를 만들었습니다

    이것이 바로 "센서로 세상을 읽는다"는 것입니다! 🌍

    핵심 정리¶

    flowchart LR A["💨 가스 분자<br/>(아날로그)"] --> B["🔬 MQ2 센서<br/>저항 변화 → 전압"] B --> C["⚡ ADC 변환<br/>0 ~ 65535"] C --> D["🖥️ print 출력<br/>숫자로 확인"] D --> E["📈 Thonny 플로터<br/>그래프 시각화"] style A fill:#FFE0B2,stroke:#F57C00 style B fill:#FFCCBC,stroke:#E64A19 style C fill:#B3E5FC,stroke:#0288D1 style D fill:#C8E6C9,stroke:#388E3C style E fill:#D1C4E9,stroke:#7B1FA2

    오늘의 핵심 코드 3줄:

    gas_sensor = ADC(Pin(26))         # 1. 센서 준비
value = gas_sensor.read_u16()     # 2. 값 읽기 (0~65535)
print(value)                      # 3. 출력 (+ 플로터 시각화)

    오늘 배운 핵심 용어:

    용어 한줄 정의
    아날로그 신호 연속적으로 변하는 신호 (가스 농도, 온도 등)
    디지털 신호 끊어진 값만 가지는 신호 (0과 1, ON/OFF)
    ADC 아날로그를 디지털 숫자로 바꾸는 변환기
    이동 평균 여러 번 읽어서 평균 내는 노이즈 저감 기법
    노이즈 센서 값에 섞인 불필요한 잡음

    📋 형성 평가 — 종합 퀴즈¶

    Q1. 다음 중 아날로그 신호의 특성으로 올바른 것은?

    ① 값이 0 또는 1만 가진다 ② 연속적으로 변하는 값을 가진다 ③ 컴퓨터가 직접 처리할 수 있다 ④ 항상 정수값만 출력한다

    정답 확인 **정답: ②** **해설**: 아날로그 신호는 시간에 따라 **연속적으로 변하는** 특성을 가집니다. ①은 디지털 신호의 특성, ③은 틀림(ADC 변환이 필요), ④도 틀림(아날로그는 소수점 포함 무한한 값 가능)입니다. 잘 맞혔다면 1단계 개념을 확실히 이해한 거예요! 👏

    Q2. Pico에서 gas_sensor.read_u16()을 호출했을 때 반환값이 0이라면, 이것이 의미하는 것은?

    ① 센서가 고장났다 ② 코드에 에러가 있다 ③ ADC 입력 전압이 0V에 가깝다 ④ USB 연결이 끊어졌다

    정답 확인 **정답: ③** **해설**: `read_u16()`의 0은 ADC에 입력되는 전압이 **0V에 가깝다**는 의미입니다. 센서 고장일 수도 있지만, 단순히 값이 0인 것만으로는 고장이라 단정할 수 없어요. 케이블이 빠져있거나, 센서가 아직 예열 중일 수도 있습니다. **데이터를 해석할 때는 여러 가능성을 고려해야 합니다!**

    Q3. 이동 평균 필터에서 count를 10에서 50으로 늘리면 어떤 변화가 일어나나요?

    ① 그래프가 더 들쭉날쭉해진다 ② 그래프가 더 부드러워지지만 반응이 느려진다 ③ 센서 정확도가 떨어진다 ④ 메모리 오버플로우가 발생한다

    정답 확인 **정답: ②** **해설**: 평균 횟수를 늘리면 노이즈가 더 많이 줄어들어 **그래프가 부드러워집니다**. 하지만 50번 읽는 데 약 500ms가 걸리므로 **실시간 반응이 느려지는 트레이드오프**가 있습니다. ①은 반대 현상, ③은 오히려 노이즈 면에서 정확도가 올라감, ④는 이 수준에서는 발생하지 않습니다. 3단계의 연습 문제를 잘 풀었다면 이해가 됐을 거예요!

    Q4. (빈칸 채우기) MQ2 센서 내부에는 200~300℃로 가열하는 ______와(과) 가스 분자에 반응하여 저항이 변하는 __________ 감지층이 있다.

    정답 확인 **정답: 히터, 금속산화물(SnO₂)** **해설**: MQ2의 두 핵심 부품입니다. **히터**가 센서를 고온으로 가열해야 **금속산화물(산화주석, SnO₂)** 층이 가스 분자에 반응할 수 있습니다. 이 때문에 예열 시간이 필요한 거예요! 1단계 MQ2 작동 원리 부분을 다시 확인해보세요.

    Q5. (서술형) 같은 교실에서 MQ2 센서를 사용했는데, A 학생 자리에서는 평상시 값이 10,000이고 B 학생 자리에서는 15,000입니다. 이런 차이가 발생하는 이유를 2가지 이상 서술하세요.

    모범 답안 확인 **모범 답안 예시**: 1. **환기 상태 차이**: A 학생이 창문 가까이 앉아 있다면 외부 공기가 잘 유입되어 가스 농도가 낮을 수 있고, B 학생이 교실 안쪽이라면 공기 순환이 원활하지 않아 배경 농도가 높을 수 있다. 2. **센서 개체 차이**: MQ2 센서는 저가 부품이라 개체마다 약간의 특성 차이(제조 편차)가 있을 수 있다. 같은 환경에서도 센서마다 출력값이 다를 수 있다. 3. **예열 상태 차이**: 한쪽 센서가 더 오래 예열되었다면 안정된 값을 보여주고, 다른 쪽은 아직 안정화 중일 수 있다. 4. **주변 환경 요인**: B 학생 근처에 손소독제, 음식물, 또는 다른 유기 물질이 있을 수 있다. **핵심 메시지**: 센서 데이터는 **절대적 수치가 아니라 상대적 변화**에 주목해야 합니다. 이것이 바로 오늘 비판적 사고 섹션에서 다룬 내용이에요! 환경, 센서 특성, 측정 조건이 모두 데이터에 영향을 미칩니다.
